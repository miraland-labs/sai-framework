# SAI Framework Roadmap

This document outlines the planned development of the Sustainable AI Intelligence (SAI) Framework specification.

## Current Status (v0.2 — July 2026)

**Completed in v0.2:**

- Precise X, Y, Z mathematical definitions with worked example
- Mandatory benchmark suite with version locks and normalization
- Four workload categories with specific weights
- Three-tier energy measurement system
- SII formula with ×100 scaling factor
- Reproducibility and conformance requirements
- Reference SII calculator (stdlib-only, informative)
- Standard evaluation report schema

**Specification artifacts:**

- README.md, SPEC.md, CONTRIBUTING.md
- BENCHMARK_GUIDE.md, MEASUREMENT_PROTOCOL.md, CONFORMANCE.md
- schemas/evaluation_report.yaml
- examples/sii_calculator.py, examples/worked_example.md

## Near-Term Goals (v0.3 — Q4 2026)

- Web3 integration specification (on-chain schema, oracle design)
- Conformance checklist validation tooling (may live in a companion repository)
- Baseline dataset publication (GPT-3.5-turbo reference values)
- Community benchmark validation working group

## Medium-Term Goals (v1.0 — 2027)

- Stable specification (v1.0)
- Formal governance model and neutral foundation transition
- SAI-Certified program with independent lab verification
- Carbon footprint certification pathway

## Long-Term Vision (2027+)

- Potential submission to standards bodies (Linux Foundation, IEEE, ISO)
- Widespread adoption in enterprise AI procurement and DeAI projects
- Ecosystem of conformant implementations, dashboards, and on-chain score registries (external to this repo)

## How to Influence the Roadmap

- Open an Issue with the label `roadmap`
- Participate in Discussions
- Submit Pull Requests for specification changes
- Join community calls (to be scheduled Q4 2026)

We welcome feedback on priorities — especially from practitioners in AI operations, sustainability, and DeAI.
